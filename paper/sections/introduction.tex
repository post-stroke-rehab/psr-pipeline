\section{Introduction}

Finger-intent prediction from sEMG is a clinically meaningful interface problem because it can connect residual forearm muscle activity to assistive control, therapy feedback, and staged post-stroke motor retraining~\citep{zheng2021semg}. In practical terms, the task is to recover a five-finger activation pattern from multichannel forearm sEMG so that downstream rehabilitation software or hardware can react to a user's intended movement.

The technical difficulty is not only model accuracy, but also pipeline reproducibility. Post-stroke sEMG is noisy, weakly structured, and sensitive to subject-specific recording conditions, so reported performance depends heavily on filtering, denoising, segmentation, label construction, and data splitting. Public repositories often expose only part of that path, which makes it hard to compare alternative predictors under a consistent signal-processing and evaluation protocol.

\repo\ addresses that gap by organizing the full workflow in one codebase~\citep{psrrepo2026}. The repository combines gesture-label harmonization, preprocessing modules, feature extraction, tensor adapters, dataset loaders, three predictor families, and saved benchmark metrics. It also includes additional two-stage healthy-to-impaired training scripts and summary files, while the two currently open pull requests extend the CNN branch with a cleaner 1D training stack, teacher-student distillation, threshold tuning, and latency benchmarking~\citep{psrpr56cnn2026,psrpr59distill2026}. Those extensions are relevant to the system story, but they are still treated separately from the merged benchmark evidence in this manuscript.

The central story of this paper is therefore systems-oriented rather than novelty-driven. We do not claim a new decoding architecture. Instead, we document a reusable benchmark workflow that makes the following engineering choices explicit: a fixed preprocessing specification, a shared window-level representation for model comparison, patient-level data splitting to reduce leakage, and repository-local metric artifacts that can be audited after training.

This updated paper makes five contributions. First, it formalizes the current preprocessing and feature pipeline implemented in the repository. Second, it explains how the code standardizes inputs so that LSTM, CNN, and GNN models can be compared inside one training and evaluation framework. Third, it consolidates saved metrics from all three predictor families, including the repository's existing LSTM artifacts that are not highlighted in the top-level README table. Fourth, it records the two open CNN-focused pull requests as under-review system extensions, covering modular 1D training, distillation, threshold tuning, and latency benchmarking without overstating them as merged benchmark evidence. Fifth, it reports the present scope limits of the benchmark so that future work can extend the repository with clearer ablations, stronger transfer studies, and broader rehabilitation validation.
