\section{Discussion}

The strongest conclusion supported by the present evidence is that \repo\ is already valuable as a reproducible benchmarking scaffold even though it is not yet a submission-ready algorithm paper. The preprocessing code, feature representation, and metric artifacts are detailed enough for another engineer or researcher to rerun the benchmark logic, inspect trade-offs, and extend the model suite without reconstructing the full pipeline from scratch.

The benchmark itself reveals two useful modeling lessons. First, a straightforward recurrent model is hard to dismiss in this setting: the saved LSTM run provides the best exact-match decoding quality and the strongest macro AUROC, which are attractive traits for downstream control. Second, the graph branch is not redundant, because its higher recall and stronger macro AUPRC suggest that explicit cross-window message passing may recover positive finger activations that the other models miss.

The repository also hints at a broader transfer-learning direction. The \texttt{run\_two\_stage.py} script and \texttt{results/README.md} file summarize healthy-to-impaired pretraining and finetuning results for CNN and LSTM branches. Those artifacts are promising, especially because rehabilitation deployments often face limited impaired-arm data, but they are packaged differently from the main benchmark tables. For that reason, this paper mentions them as follow-on evidence rather than merging them into the headline comparison.

Several limitations remain. The benchmark currently covers one dataset, one main processed benchmark path, and no stored ablation study for preprocessing or architecture variants. The GNN implementation uses dense window graphs, which may become expensive as sequence length grows. The repository also includes a Mamba notebook for signal translation, but that component is not integrated into the same evidence-complete evaluation path as the direct finger-intent predictors. These gaps do not invalidate the current benchmark, but they do define the most important next steps for turning \repo\ from a strong engineering base into a stronger empirical paper.
