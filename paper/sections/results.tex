\section{Results}

The current repository benchmark exposes a clear trade-off between balanced sequence modeling and high-recall graph modeling. Table~\ref{tab:model-comparison} shows that the LSTM achieves the strongest exact-match accuracy, finger accuracy, and macro AUROC, while the GNN narrowly leads on macro F1 and macro AUPRC. The CNN remains competitive as a lightweight baseline but does not surpass the best LSTM or GNN score on any headline metric in the saved artifacts.

The gap between the GNN and the LSTM is easiest to interpret through precision-recall behavior. The GNN posts the highest macro recall among the saved runs (0.832 versus 0.768 for the LSTM and 0.697 for the CNN), which lifts its macro F1 and AUPRC. The LSTM, however, converts that lower recall into materially stronger exact-match behavior and substantially stronger AUROC, suggesting a more balanced discrimination profile across thresholds.

Open PR evidence suggests that the CNN branch may move closer to the LSTM and GNN frontier once distillation utilities are merged. PR \#59 reports a provisional CNN-Micro student distilled from a CNN teacher with macro F1 0.7096, finger accuracy 0.7990, macro AUROC 0.8662, and macro AUPRC 0.7838, improving over the pull request's stated CNN baseline of 0.6817, 0.7750, 0.8559, and 0.7319 while keeping the student at 51{,}493 parameters~\citep{psrpr59distill2026}. Because those numbers currently live in an open pull request rather than the committed `metrics/` tree, we report them here as provisional extension evidence instead of merging them into Table~\ref{tab:model-comparison}. The same PR also records local CPU proxy latency of 0.274 ms per sample on average with a 0.505 ms p95, which is useful for screening edge candidates but is not a substitute for hardware-validated rehabilitation timing.

Finger-wise scores show that the ranking is not uniform across all outputs. Table~\ref{tab:per-finger-f1} indicates that the GNN is strongest on the thumb label, while the LSTM is strongest on the index, middle, ring, and little finger labels. This pattern is consistent with the repository-wide picture: graph aggregation appears to favor aggressive positive detection, whereas the recurrent model distributes performance more evenly across fingers.

One additional paper-level update is worth noting. The top-level repository README foregrounds the GNN and CNN comparison, but the repository also contains complete LSTM metric artifacts under `metrics/lstm/`. Including those saved LSTM results materially changes the interpretation of the benchmark, because the best overall system is no longer a simple GNN-versus-CNN choice. At the same time, the open CNN PRs show that this ranking should not be interpreted as final: the repository is still actively evolving toward a lighter-weight distilled branch with stronger deployment-facing evidence.
